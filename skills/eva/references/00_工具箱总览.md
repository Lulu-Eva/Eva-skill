# Eva-skill 思维流爆款短视频工具箱总览

更新时间：2026-04-28

本目录把“思维流口播爆款方法论”拆成一个主入口和六个公开子 skill。结构参考 `/dbs` 工具箱：主入口判断用户需求并路由，子 skill 各自完成一个明确任务。

## 方法来源说明

本工具箱基于“思维流口播爆款方法论”和相关短视频课程经验提炼而成，只保留可迁移、可公开使用的短视频生产规则。

公开版不包含原始课程逐字稿、本地文件路径、未公开的内部资料和私有业务流程。

## 为什么是 6 个公开子 skill

这套方法论的生产链路是：

`选题上限` -> `拆解网感` -> `标题封面点击` -> `正文留存` -> `口播表现` -> `数据复盘`

因此公开版只保留这六个模块，聚焦“单条思维流短视频能不能爆”。

## 核心世界观

1. 选题和标题决定 80%-90% 的数据上限。
2. 新手阶段不要原创标题和选题，先借用已验证的爆款结构。
3. 小红书是一个巨大的编辑器，创作者的工作是搜索、筛选、拼接、改编。
4. 点赞的本质是“我被看见了”，尤其是弱者处境被说中。
5. 标题封面是广告位，不是作文题目，目的是让用户一眼点进来。
6. 正文的任务是顺着用户疑问自圆其说，不要自嗨表达。
7. 口播不是朗读，是“和闺蜜分享一件事”。

## 共用质量规则

1. **先判断能不能继续做。** 如果选题、素材、正文或数据不足以支撑当前任务，先指出缺口，不要硬生成。
2. **所有标题和选题都要可追溯。** 输出时标明来源类型、验证等级、改写方式和需要用户搜索验证的关键词。
3. **不编造素材。** 不编造数据、经历、爆款来源、评论区原话、博主案例。缺素材时用 `[待补充]` 标注，或生成“可替换素材槽位”。
4. **人设信息要显性化。** 写口播稿前优先确认目标人群、账号人设、真实经历、账号价值和不能说的内容。
5. **子 skill 要会接力。** 每个子 skill 结束时，根据问题类型推荐下一步，而不是只完成本环节。

## 推荐使用顺序

1. 没有选题：先用 `01_eva-topic_爆款选题.md`
2. 有对标笔记：先用 `02_eva-deconstruct_爆款拆解与网感训练.md`
3. 有选题但点击弱：用 `03_eva-title-cover_标题封面点击率.md`
4. 有标题要写稿：用 `04_eva-script_思维流口播文案.md`
5. 有稿子要拍：用 `05_eva-performance_口播情绪节奏.md`
6. 已发布或要批改：用 `06_eva-review_发布复盘诊断.md`
